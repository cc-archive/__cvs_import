We have a prototype search engine that:

  • gathers URLs from AllTheWeb
  • provides an easy-to-extend URL gathering API
  • uses gathered URLs to download documents, check RDF, and index the content
  • searches by license property and full text

We can now pursue development of this search engine with a few efforts, most of which can be done in parallel.

More URL Gathering

Searching through more URLs is better. We can increase the number of URLs we gather by:

  • playing tricks on AllTheWeb and Google
  • writing our own crawler or using an existing one

Automation and Production Deployment

The current system can be automated by using simple cron scripts, but this automation must be thoroughly tested. One goal would be to make this system totally automated and fully-tested, with error-reporting via email when something goes wrong.

Tuning the Search Application

As we gather more pages to index, we may want to tune how we index various pieces of the page to provide better searching capabilities, advanced searching capabilities (by author), etc..

Load-Testing the Search

This is crucial before we deploy at any level. It's also the most complicated aspect of the whole thing. The search is different from the rest of the CC site in that:

  • it provides an active, dynamic service
  • it will get significant usage, increasing with time
  • it will require significant hardware resources, much more so than anything else the CC site currently does.
